次世代クラウド環境 構築ロードマップ

本資料は、ゼロトラスト環境構築に向けた全体ロードマップと、
プロジェクトを成功に導くための重要な補足事項（役割分担・事前準備）をまとめたものです。

私たちが目指す3つのゴール（基本方針）

単なるツールの導入ではなく、以下の3つの制御を効かせることで「利便性」と「安全性」を両立させます。

👤

1. ID認証の徹底

すべてのシステムの入り口をGoogleアカウントに統一します。パスワードに加え「二段階認証」を必須とし、悪意のあるなりすましを入り口で完全に遮断します。

💻

2. デバイス認証

会社が管理・許可しており、かつパスコード設定などのセキュリティが担保された健全な端末（MDM管理下）からのみ、データへのアクセスを許可します。

🛡

3. 権限管理と監視

業務に必要な最小限のデータにのみアクセス権を付与します。クラウド上の動きを常に監視し、機密情報の社外持ち出しなどの危険な操作をシステムで自動ブロックします。

全体ロードマップ：8段階の導入ステップ

プロジェクト全体の「地図」です。上から下へ、各段階で何が実現し、何を決定していくかを確認できます。

1

ステップ1: ログイン画面のGoogle化

古い社内サーバー（AD）の利用を停止し、パソコンを開くパスワードをGoogleアカウントに統一します。

実現するメリット

  • ✓ 場所を問わずPCにログイン可能に
  • ✓ 管理者層への「二段階認証」先行導入

当ステップの決定・準備事項

  • 全端末の洗い出しと「デバイス管理台帳」の作成
  • PCのローカル管理者パスワードの全台確認
  • 二段階認証の必須化期限・パスワードルールの決定
2

ステップ2: 端末の管理強化・紛失対策

モバイル管理機能(MDM)を使い、会社支給のスマホやPCを「会社の管理下」に置き、安全を担保します。

実現するメリット

  • ✓ スマホ紛失時の遠隔データ消去（リモートワイプ）
  • ✓ 業務に不要/危険なアプリのインストール制限

当ステップの決定・準備事項

  • 対象者のApple IDパスワード確認と「探す」のオフ
  • USBメモリの利用方針（全面禁止か一部許可か）
  • スマホでの利用許可/禁止アプリの選定
3

ステップ3: 社内データのクラウド完全移行

壊れやすい社内ファイルサーバーを廃止し、Googleドライブへすべての業務データを移行します。

実現するメリット

  • ✓ 自宅や出張先からでも安全に社内ファイルへアクセス
  • ✓ サーバーの故障リスクや買い替え費用の完全な撤廃

当ステップの決定・準備事項

  • 新Googleドライブの「フォルダ階層」と「アクセス権限」の設計
  • 複合機の「スキャンデータ保存先」の新しい運用ルールの決定
  • 旧ファイルサーバー（NAS）の最終的な廃止時期
4

ステップ4: ブラウザの集中管理と拡張機能制御

業務で利用するChromeブラウザを会社で一括管理し、利便性向上とWeb経由の脅威からの保護を両立させます。

実現するメリット

  • ✓ Endpoint管理拡張機能の強制インストールによる端末の集中管理
  • ✓ 許可されていない拡張機能の制限・私用ログインの禁止

当ステップの決定・準備事項

  • 業務上必要な「許可する拡張機能」のリストアップ
  • 全社員に自動配布する「標準ブックマークリスト」の作成
5

ステップ5: 全社システムのパスワード共通化

確立した安全な土台の上で、社内の様々なシステム（SaaS）のログインをGoogleに統合（SSO）します。

実現するメリット

  • ✓ 社員の「パスワード忘れ」による業務停止の解消
  • ✓ 退職時のアカウント停止作業のワンクリック化

当ステップの決定・準備事項

  • 連携対象となる各SaaSの「管理者アカウント権限」の確保
  • 共通化を進めるシステムの「優先順位」と年間スケジュールの承認
6

ステップ6: 情報漏洩の自動ブロック

機密情報を社外へ出そうとした際、システムが自動で検知してブロックする仕組み（DLP）を導入します。

実現するメリット

  • ✓ 機密情報（マイナンバー等）の外部送信ブロック
  • ✓ 指定外の端末（個人スマホなど）からのアクセス拒否

当ステップの決定・準備事項

  • システムに検知させるべき「機密キーワード」の設定
  • 本格ブロック前の「監査（テスト）期間」のスケジュール合意
7

ステップ7: ルール運用と定期点検

構築した安全な環境を陳腐化させないため、継続的に監視し、運用ルールを定着させます。

実現するメリット

  • ✓ 未許可クラウドサービス（シャドーIT）の検知
  • ✓ 退職者アカウント等の消し忘れ防止

当ステップの決定・準備事項

  • 新入社員・退職者発生時の「アカウント・端末準備マニュアル」の作成
  • 外部への共有リンク等の定期チェック（棚卸し）頻度の決定
8

ステップ8: 高度なWebガバナンス管理

Chrome Enterprise（有償ライセンス）を活用し、ブラウザを境界とした強力なセキュリティと詳細な監査環境を構築します。

実現するメリット

  • ✓ 高度なURLフィルタリングとマルウェア・フィッシング防御
  • ✓ 誰が・いつ・どのデータを操作したかの詳細な監査ログの取得

当ステップの決定・準備事項

  • Chrome Enterpriseライセンス（Core / Premium等）の調達方針
  • 取得した監査ログの定期的なレビュー体制の構築
